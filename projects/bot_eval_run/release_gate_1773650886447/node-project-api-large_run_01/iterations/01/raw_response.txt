It uses in-memory repositories and follows a modular structure under `src/modules/`.\n\n## Features\n- In-memory data storage\n- Modular architecture with separate modules for projects, tasks, members, and comments\n- RESTful API endpoints for CRUD operations\n- Error handling with consistent payload shape `{ error: { code, message } }`\n\n## Getting Started\n1. Clone the repository.\n2. Install dependencies using `npm install`.\n3. Run the application using `node src/server.js`.\n\n## Endpoints\n- **GET /health**: Check the health of the API.\n- **POST /projects**: Create a new project.\n- **GET /projects**: List all projects.\n- **GET /projects/:projectId**: Get a specific project by ID.\n- **POST /projects/:projectId/members**: Add a member to a project.\n- **GET /projects/:projectId/members**: List all members of a project.\n- **POST /projects/:projectId/tasks**: Create a new task for a project.\n- **GET /projects/:projectId/tasks**: List all tasks of a project.\n- **GET /projects/:projectId/tasks?status=done**: List only completed tasks of a project.\n- **PATCH /projects/:projectId/tasks/:taskId**: Update the status of a task.\n- **POST /projects/:projectId/tasks/:taskId/comments**: Add a comment to a task.\n- **GET /projects/:projectId/tasks/:taskId/comments**: List all comments of a task."
